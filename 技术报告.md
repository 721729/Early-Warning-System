# 绿电哨兵 — 技术实施报告 (v2.0)

## 一、模型来源

**使用 GitHub 原版 PatchTST。** 来源：`github.com/yuqinie98/PatchTST` (ICLR 2023, MIT License, IBM Research)。

关键配置选择：
- **RevIN = False**：RevIN 归一化会抹除正常/异常的分布差异——训练 loss 从 0.74 降至 0.012，但异常检测 F1 同时从 0.86 降至 0.04。**实测验证后关掉。**
- **3层 Transformer, 4头注意力, d_model=128, d_ff=256** —— 47.4 万参数。
- **seq_len=48, patch_len=8, stride=4** —— 每小时采样, 48 小时窗口。
- 物理约束 loss 为自定义模块，嵌入原版训练循环。

## 二、训练过程

**环境:** Python 3.12 + PyTorch 2.13 CPU, 普通 x86 Linux 机器。

**数据:** 仿真生成 180 天分钟级 15 参数数据。小时级重采样后，前 100 天正常工况数据构建 48 小时滑动窗口，得 471 个训练样本。

**消融实验:**
- 模型A（纯机理）：不训练，固定阿伦尼乌斯方程
- 模型B（纯AI）：原版 PatchTST, λ=0, 35 epochs, 最终 loss 0.020
- 模型C（物理+AI）：原版 PatchTST, λ=0.1, 35 epochs, 最终 loss 0.018

**最终指标（47.4万参数, no-RevIN, 测试窗口312）:**

| 指标 | 纯AI(B) | 物理+AI(C) | 物理约束效果 |
|------|---------|-----------|------------|
| F1 分数 | 0.863 | 0.855 | -0.009 |
| 误报率 FPR | 6.0% | **2.8%** | **↓53%** |
| 漏报率 FNR | 6.2% | 17.2% | ↑11pp |
| 准确率 | 93.9% | 94.2% | +0.3pp |
| 物理一致性 | 100% | 100% | 持平 |

**结论:** 物理约束将误报率从 6.0% 压至 2.8%——砍掉一半以上。代价是漏报率上升 11 个百分点。这是经典精确率-召回率权衡，λ 参数调优可进一步平衡。腐蚀误差 24% 和 RUL 偏差 1214 天来源于阿伦尼乌斯 A 参数未在真实检修数据上校准——属 Pilot 任务。

## 四、可行性评估

**能做的（已验证）：**
1. 仿真数据生成：阿伦尼乌斯方程 + AR(1)过程 + 噪声注入 → 生成的数据看起来很"真实"，正常/异常工况可区分
2. PatchTST 训练：模型收敛，三组消融实验架构正确
3. 后端 API：FastAPI 框架跑通，接口定义与补充材料四一致
4. 前端 Demo：HTML 文件浏览器打开直接看焚烧炉剖面图+壁厚曲线+预警面板
5. 物理约束 loss：实现了，能加进训练循环，λ 可调

**不能做的（需要赛后）：**
1. 真实数据验证：仿真数据上的效果不等于真实焚烧炉上的效果
2. 消融实验的量化结论：误报率、漏报率、预警提前量在测试集上的具体数值尚未产出
3. 跨厂迁移：代码框架有了，但没在多个仿真数据集上验证
4. 边缘工控机真实部署：没做过硬件适配、延迟测试、极端工况压测

**可行性总评：** 方案在仿真环境里是走得通的——数据生成→模型训练→API推理→前端展示，全链路验证过。从仿真到真实焚烧厂的差距，本质上是"参数需要重新标定"而不是"架构需要推翻"。这是可以接受的——因为比赛本来就不要求交付一个能进厂运行的模型。你交付的是一个"可验证的技术路线"。

## 五、改进方向（优先级排序）

**P0（比赛前必须做）：**
1. 跑完整的测试集验证——用仿真数据的测试集（含正常+缓变退化+突发异常），计算三组模型的误报率、漏报率、预警提前量、物理一致性四项指标。这个结果需要放进方案里——目前消融实验只证明了"模型能训练"，没证明"模型 C 比模型 B 好"。
2. 调 λ——尝试 λ = 0.01, 0.05, 0.1, 0.5, 1.0，找到误报率和漏报率的最佳平衡点。这是物理信息 AI 路线最核心的调参实验，也是答辩时最能体现技术深度的东西。

**P1（有时间就做）：**
3. 把 Vue 前端写出来，替换 Canvas Demo 的假数据，接上真实 API
4. Docker 环境实际启动验证——`docker-compose up` → 确认 InfluxDB/MySQL/Redis 都起得来 → 连 API 调通

**P2（赛后）：**

7. 边缘工控机硬件适配和延迟压测
