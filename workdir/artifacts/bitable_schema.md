# 飞书多维表格 Schema（复赛 PoC 计划）

后端 `/api/v1/feishu/sync-alert` 端点会向下列 3 张多维表格写入数据。

## 表 1：设备健康度（核心看板）

| 字段 | 类型 | 说明 |
|------|------|------|
| 电厂 | 单选 | 高能旗下 13 座厂 |
| 设备 | 单选 | 6 设备：入口→出口→中温→低温→省煤器→引风机 |
| 当前壁厚(mm) | 数字 | 仿真生成 / 未来 DCS 真实值 |
| 健康度(%) | 公式 | =(当前壁厚 - 3.0) / 3.0 × 100 |
| RUL(天) | 数字 | 剩余寿命 |
| AI 状态 | 单选 | 绿/黄/橙/红 |
| 最后更新时间 | 时间类型 | |

## 表 2：预警记录

| 字段 | 类型 | 说明 |
|------|------|------|
| 关联设备 | 链接到表 1 | |
| 预警等级 | 单选 | 黄/橙/红 |
| 预警时间 | 时间类型 | |
| HCl 浓度 | 数字 | mg/m³ |
| 腐蚀速率(mm/年) | 数字 | |
| 关联 RUL(天) | 数字 | |
| 处置状态 | 单选 | 待确认/处理中/已完成/已忽略 |
| 责任班组 | 单选 | |

## 表 3：检修工单

| 字段 | 类型 | 说明 |
|------|------|------|
| 关联预警 | 链接到表 2 | |
| 工单号 | 自动编号 | |
| 设备 | 链接到表 1 | |
| 故障描述 | 长文本 | |
| 建议方案 | 长文本 | AI + 规则引擎生成 |
| 备件消耗 | 多选 | 链接到备件库存表 |
| 责任人 | 人员字段 | |
| 状态 | 单选 | 待派工/进行中/已完成 |

## 接入方式

- **Demo 阶段**：FastAPI `/api/v1/feishu/sync-alert` → lark-cli `base +record-create`
- **Pilot 阶段**：直接换成真实 DCS 数据流
- 后端只新增 1 个 endpoint + 3 个 base token，不改动现有架构
